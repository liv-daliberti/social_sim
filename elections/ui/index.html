Bayesian Network Sampling
Each node X has a set of parents Pa(X). We find the CPT row whose conditions are a subset of the current parent states — most specific match wins (highest count of matched conditions):
P(X = x | Pa(X) = pa)
Then draw: x ~ Categorical(P).
Root nodes A1, A2, A3 have no parents and use a fixed prior P(X) directly. All 23 nodes are sampled left-to-right in topological order so every parent is already resolved before its children.
Surprise (Self-Information)
After sampling x with probability p, how unexpected was that outcome?
S(x) = −log₂ p(x) [bits]
Low p → high surprise. Thresholds used here:
< 1 bit — expected outcome
1 – 2.5 bits — moderately surprising
⚡ > 2.5 bits — highly surprising
Edge Causal Influence (TVD)
For a group edge A → B, how much does A's sampled state shift B's probability distribution? For each child node b ∈ B that has a parent a ∈ A:
TVD(P, Q) = ½ · Σₓ |P(x) − Q(x)|
1. Group CPT rows by the value of parent a
2. Average distributions within each group
3. Compute TVD for every pair of groups
4. Average all pairwise TVDs across all (a, b) pairs
Range [0, 1]: 0 = parent irrelevant  ·  1 = parent fully determines child
Computed analytically from the CPT structure — not from simulation runs. The default visual weight is 50% (uniform prior on influence). Click any edge % to override it.
Hoeffding Validation
After n = 1 000 runs, Hoeffding's inequality bounds the chance that the empirical frequency deviates from the true probability by more than ε:
P(|P̂ − p| > ε) ≤ 2 · exp(−2nε²) Setting α = 1%: ε = √(−ln(α/2) / 2n) = √(−ln(0.005) / 2000) ≈ 5.15 percentage points
The two quantities being compared:
Model marginal = average of P(X | parents) over all n runs = E[P(X|Pa)]
Empirical freq. = observed count(X = x) / n
By the law of total expectation: E[P(X|Pa)] = P(X), so both quantities converge to the same value as n → ∞. A ✗ means the sampler's output disagrees with the CPT — a bug signal, not a model finding.
News Forecast Power — Two Methods
How much of the election outcome can an observer predict knowing only E = (E1,E2,E3,E4)? Both methods compute the same quantity but differently. The gap between them is sampling noise.
Method 1 · Empirical Monte Carlo
Run N simulations from the model. Group runs by the E-values they produced. Count wins per group.
P̂(Blue | E=e) = #{runs where E=e AND Blue wins} / #{runs where E=e} acc(e) = max( P̂(Blue|e), 1 − P̂(Blue|e) ) Acc_emp = Σ_e (#{E=e}/N) · acc(e)
Cells with <5 hits are excluded (single-sample overfitting). Converges to the exact answer as N → ∞.
Method 2 · Exact Variable Elimination
Enumerate all states analytically in topological order, exploiting the DAG structure to precompute reusable partial sums.
Pass 1 — marginalize G (deterministic I1/I2): p_blue_d[b3,c2,d1,d2,d4] = Σ_{g1,g2,g3} P(G1|·)·P(G2|·)·P(G3|·)·blue(g1,g2,g3) Pass 2 — fold in D4 (Issue Salience): v[b3,c2,d1,d2,e1] = Σ_{d4} P(D4|c2,e1) · p_blue_d[·] Pass 3 — fold in D1,D2 (Momentum): u[b1,b2,c3,e3,b3,c2,e1] = Σ_{d1,d2} P(D1|·)·P(D2|·) · v[·] Enumeration — over (A,B,C,E) in topo order: w = P(a1)·P(a2)·P(a3)·P(b|a)·P(c1|a)·P(c2|c1) ·P(c3|c2)·P(c4|c2)·P(e1|c2,a1)·P(e2|c4,a2) ·P(e3|c3,e2,b1,b2,a3)·P(e4|c4,b3) joint_blue[e] += w · u[b1,b2,c3,e3,b3,c2,e1] joint_wt[e] += w Result: P(Blue|E=e) = joint_blue[e] / joint_wt[e] P(E=e) = joint_wt[e] Acc_exact = Σ_e P(E=e) · max(P(Blue|e), P(Red|e))
D3 (Voter Uncertainty) has no path to I2 and is correctly omitted. All CPT lookups are precomputed as dicts before the enumeration. Result is cached and recomputed only when CPTs change.
Observation vs intervention: Both methods compute P(I2|E=e), the observational posterior. This differs from P(I2|do(E=e)) because observing E updates beliefs about upstream A,B,C nodes that also directly influence D and G.

Bayes Ceiling — Best Possible Performance

The model only ever sees E1–E4 (news signals). The best it can ever do is predict the true conditional probability:

p*(E) = P(Blue wins | E1, E2, E3, E4)

Reward ceiling depends on label type:

Float-prob labels (current): settlement_yes = p*(E). A perfect model predicts p* exactly → Brier ≈ 0 → reward ≈ 0.5. The dashed line is at 0.5.

Binary labels: settlement_yes ∈ {0,1} (single draw). Even a perfect model has irreducible noise:

Brier_floor = E[ p*(E) · (1 − p*(E)) ] ≈ 0.241
Reward_ceiling = 2 × (0.25 − Brier_floor) ≈ 0.017

Accuracy ceiling (always meaningful): Even knowing p* exactly, the best classification accuracy (predicting Blue vs Red) given only news signals is:

Acc_ceiling = E[ max(p*(E), 1−p*(E)) ] ≈ 57.3%

This is shown as a dashed line on the accuracy chart. It represents the best any model could classify elections given only news signals — not more information can be extracted from E1–E4 for binary classification.
